## Reporting a vulnerability

Report vulnerabilities through [GitHub Security Advisories](https://github.com/MoizIbnYousaf/marketing-cli/security/advisories/new). Do not open a public issue.

Expect an initial response within 72 hours. Confirmed vulnerabilities get a fix timeline in the advisory.

## Threat model

mktg runs commands on behalf of AI agents. The agent is not a trusted operator. All inputs (skill names, brand file content, JSON payloads, path arguments) are treated as potentially adversarial.

## Input validation

Every input passes through a validation pipeline before reaching any file system or command operation. All validators return `{ ok, message }` and never throw.

### Path safety

**`sandboxPath(root, untrusted)`**: Resolves a path and verifies it stays within the project root.

- Rejects absolute paths (`/etc/passwd` ‚Üí blocked)
- Rejects traversal (`../../.ssh/id_rsa` ‚Üí blocked)
- Rejects symlink escapes (symlink pointing outside root ‚Üí blocked)
- All file writes are constrained to `brand/`, `marketing/`, and `.mktg/`

**`detectDoubleEncoding(input)`**: Catches percent-encoding bypass attempts.

- Blocks double-encoded sequences (`%252e%252e` ‚Üí would decode to `..`)
- Blocks single-encoded path characters (`%2f` ‚Üí `/`, `%5c` ‚Üí `\`, `%2e` ‚Üí `.`)
- Blocks null-byte injection (`%00`), CRLF injection (`%0a`, `%0d`), tab injection (`%09`)
- Case-insensitive; `%2E` and `%2e` are both caught

### Text inputs

**`rejectControlChars(input)`**: Strips the attack surface from text fields.

- Blocks bytes `0x00`‚Äì`0x08`, `0x0B`, `0x0C`, `0x0E`‚Äì`0x1F` (C0 controls)
- Blocks `0x7F` (DEL) and `0x80`‚Äì`0x9F` (C1 controls)
- Allows tabs (`0x09`) and newlines (`0x0A`, `0x0D`), which are needed for markdown content

**`validateResourceId(id)`**: Constrains skill names, brand file names, and command names.

- Allows only `[a-z0-9][a-z0-9._-]*`
- Rejects `?`, `#`, `%`, spaces, slashes, uppercase, and unicode
- Enforces 128-character maximum length

### JSON inputs

**`parseJsonInput(raw)`**: Parses agent-supplied JSON with safety limits.

- Enforces 64KB size limit
- Detects prototype pollution (`__proto__`, `constructor` keys ‚Üí rejected)

### Combined pipeline

**`validatePathInput(input)`**: Chains all path-relevant validators in sequence: control character rejection, double-encoding detection, then sandbox path resolution. One call, all checks.

## Exit codes

Structured exit codes (0‚Äì6) with typed error envelopes on every command. See [`docs/EXIT_CODES.md`](docs/EXIT_CODES.md).

## Scope

This policy covers the `mktg` CLI package (`marketing-cli` on npm). Chained-in tools (firecrawl, ffmpeg, playwright, whisper, etc.) have their own security policies; report vulnerabilities in those tools to their respective maintainers.

## Disclosure history

### 2026-04-16: Next.js preview keys in committed `website/.next/` build artifacts

Commit `2f952f4` accidentally committed a `website/.next/` build cache snapshot containing six auto-generated Next.js preview-mode keys (HMAC + AES) used to sign Next.js `<PreviewMode>` cookies for the `website/` Next.js subdirectory. The artifacts were reverted in `91794de` ("chore: drop accidentally committed website/.next + website/out artifacts").

The keys are dead build artifacts:
- They were generated by Next.js per build, never set by the maintainer.
- The `website/` subdirectory containing the Next.js app was never deployed under those keys at this repo's domain.
- The keys do not protect any active service.
- The keys remain reachable via `git log --all` for historical commits but are not referenced by any current build, deployment, or runtime.

No rotation is required because the keys never gated a live system. The history is left intact rather than rewritten because (a) the keys are inert, (b) `git filter-repo` would invalidate already-cloned SHAs without reducing risk, and (c) published `marketing-cli` npm tarballs do not include the `website/` directory in any case.

If you discover a concrete exploit path that depends on these dead keys, please open a private security advisory per the reporting flow above.
