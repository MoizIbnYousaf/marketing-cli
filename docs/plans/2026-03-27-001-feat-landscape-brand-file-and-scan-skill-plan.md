# feat: Add landscape.md as 10th Brand File + /landscape-scan Skill

## Overview

The CMO skill generates public-facing content based on brand files that are static research snapshots. This causes it to make confidently wrong claims about the ecosystem — like saying "people manually copy skills from GitHub" when tools like `skills.sh` exist with `npx skills add`. There is no ground-truth document about what's actually happening in the ecosystem RIGHT NOW.

**The fix:** A new brand file `brand/landscape.md` written by a new skill `/landscape-scan` that chains `/last30days` for live research, validates findings with the user, and produces a Claims Blacklist that hard-gates content generation.

## Problem Statement

Content skills (direct-response-copy, seo-content, social-campaign, etc.) currently have no mechanism to verify ecosystem claims before making them. The CMO can read competitors.md for deep competitor profiles and positioning.md for differentiation, but neither captures real-time ecosystem state: what tools exist, what people actually use them for, what has changed in the last 30 days. This gap produces confidently wrong marketing content — the worst kind of marketing.

## Proposed Solution

1. Add `landscape.md` as the 10th brand file in the type system, templates, and all command pipelines
2. Create `/landscape-scan` skill that chains `/last30days` for live research + user validation
3. Introduce a **Claims Blacklist** as a hard gate — if a claim is blacklisted, no content skill may make it
4. Integrate into CMO activation protocol so landscape freshness is checked before routing to content skills

## Technical Approach

### Architecture

The change is surgical but wide — it touches the type system which cascades to every command. The key insight from research: `BRAND_FILES` in `src/types.ts` is the single source of truth. Doctor, init, and scaffolding all iterate this array automatically. Only plan, status, and context have their own hardcoded lists that need manual updates.

#### Phase 5: New Skill — `/landscape-scan`

**Files:** `skills/landscape-scan/SKILL.md`, `skills/landscape-scan/references/query-templates.md`, `skills/landscape-scan/references/landscape-schema.md`

**Tasks:**
- [ ] Create `skills/landscape-scan/SKILL.md` following competitive-intel structure:

```markdown
---
name: landscape-scan
description: >
  Scan the current market landscape and produce a ground-truth ecosystem
  snapshot. Chains /last30days for live research, validates with user,
  and writes brand/landscape.md with a Claims Blacklist that hard-gates
  all content generation. Use when: "landscape", "ecosystem", "market
  snapshot", "ground truth", "what's happening", "refresh landscape",
  "market trends", or before any content campaign to verify claims.
---
```

**Orchestration flow (6 phases):**

1. **Preflight** — Check brand/ exists, read competitors.md + positioning.md + audience.md for context. Detect create vs. refresh mode (landscape.md exists?).

2. **Build research query** — Extract product name, category, competitor names from brand files. Construct focused query: `"{product} {category} {competitor1} {competitor2} ecosystem tools trends"`.

3. **Run /last30days research** — Invoke Skill tool → `last30days` with constructed query. Parse findings for: key developments, tool launches, competitor moves, sentiment shifts. **Fallback:** If /last30days unavailable, use WebSearch for equivalent coverage.

4. **Validate with user (AskUserQuestion)** — Generate 3-5 targeted questions from delta between research findings and existing brand files. Cap at 5 questions. Prioritize contradictions over confirmations. Example: "Research shows [X] now offers [feature]. Your competitors.md doesn't mention this. Accurate?"

5. **Synthesize into landscape.md** — Write all sections per SCHEMA.md contract. Build Claims Blacklist from contradictions. Build Claims Confirmed from validations. Flag Brand File Contradictions with recommended fix actions.

6. **Cross-reference** — Compare findings vs. positioning.md, competitors.md, audience.md. Flag stale data in other files. Suggest specific refresh skills (e.g., "Run /competitive-intel to update competitor profiles"). Don't auto-fix — those files have owner skills.

- [ ] Create `skills/landscape-scan/references/query-templates.md` — research query templates for different industries
- [ ] Create `skills/landscape-scan/references/landscape-schema.md` — full output schema with examples

**Anti-patterns (with WHY):**
- Don't auto-update other brand files → WHY: Each file has an owner skill. Cross-updating creates ownership conflicts and breaks the "skills never call skills" principle.
- Don't skip user validation → WHY: Research can surface false positives. The user is the ground truth validator. Without validation, the Claims Blacklist could incorrectly block valid claims.
- Don't run without brand context → WHY: Without competitors.md and positioning.md, the research query will be unfocused and return generic results. L0 mode asks 3-5 questions to build context first.

## System-Wide Impact

### Interaction Graph

Adding `"landscape.md"` to `BRAND_FILES` in types.ts triggers:
1. `scaffoldBrand()` in brand.ts → creates landscape.md on `mktg init`
2. `getTemplateHashes()` in brand.ts → auto-computes SHA-256 for template detection
3. `isTemplateContent()` → detects if landscape.md is still scaffold
4. `checkBrand()` in doctor.ts → auto-includes in health checks (iterates BRAND_PROFILE_FILES)
5. `plan.ts` → shows "Populate landscape.md" when template detected (via foundationOrder + skillMap)
6. `status.ts` → shows landscape.md state in nextActions (via foundationFiles + skillMap)
7. `context.ts` → includes in context compilation (via FILE_PRIORITY + CONTEXT_MATRIX)
8. CMO activation → checks freshness, reads Claims Blacklist before routing to content skills

### Error & Failure Propagation

- If /last30days skill is unavailable: landscape-scan falls back to WebSearch. Graceful degradation.
- If user skips validation questions: landscape.md is written with unvalidated claims. Claims Blacklist may have false positives. This is acceptable — better to over-block than under-block.
- If landscape.md is stale (>14 days): CMO warns but doesn't block. Content generation continues with stale data + warning.

### State Lifecycle Risks

- **No orphan risk:** landscape.md is a file on disk. Partial write = incomplete file. Re-running /landscape-scan overwrites completely.
- **No cache staleness:** Template detection uses SHA-256 at read time, not cached.
- **Cross-file consistency:** landscape.md may contradict competitors.md or positioning.md. The Brand File Contradictions table explicitly surfaces this. Resolution is manual (user runs the owner skill).

### API Surface Parity

All CLI commands that iterate brand files will automatically pick up landscape.md:
- `mktg init` — auto-scaffold via BRAND_FILES
- `mktg doctor` — auto-check via BRAND_PROFILE_FILES
- `mktg brand freshness` — auto-include in freshness report
- `mktg brand export/import` — auto-include in export bundle
- `mktg status` — needs manual update (hardcoded foundationFiles)
- `mktg plan` — needs manual update (hardcoded foundationOrder + skillMap)
- `mktg context` — needs manual update (hardcoded FILE_PRIORITY)

## Key Design Decisions

1. **landscape.md is a PROFILE file** (not append-only) — it gets overwritten on each scan, has freshness checks, template detection. 14-day freshness is handled by `review_interval_days` in the manifest, not by file type.
2. **14-day freshness** — fastest of any brand file. Ecosystems move fast.
3. **Claims Blacklist is a HARD GATE** — overrides all other brand files. This is the core value proposition.
4. **Chains /last30days, doesn't replace it** — /last30days is an independent skill. landscape-scan orchestrates it.
5. **NOT in creative layer** — visual skills (paper-marketing, creative, app-store-screenshots) don't need ecosystem data. Saves tokens.
6. **No auto-populate from URL scraping** — `mktg init --from <url>` does NOT fill landscape.md. Landscape requires active live research, not HTML extraction.
7. **User validation is mandatory** — research alone isn't enough. AskUserQuestion confirms findings before blacklisting claims.
8. **landscape.md != competitors.md** — competitors = deep profiles of specific players. landscape = current ecosystem truth including tools, trends, benchmarks, and verified/debunked claims.
