# Email 05 — Compounding loop

**Send:** Day 8  
**Subject:** The habit that makes mktg worth keeping  
**Preview:** learnings.md, assets.md, document-review.

```
Subject: The habit that makes mktg worth keeping

Week-one installers usually fail in one of two ways:

1) They never fill brand/ — every skill stays generic
2) They generate piles of drafts — and never log what worked

The compounding loop is small:

After every real marketing action:
- Append a learning to brand/learnings.md (what broke, what converted, what to stop saying)
- Append shipped assets to brand/assets.md (path, type, skill)
- Re-run /document-review when files feel stale or inconsistent
- Update voice/positioning when you catch drift — then regenerate the artifact

That is how brand memory compounds across sessions. Not by magic. By files your agent can read tomorrow.

Keep the north star boring:
npm i -g marketing-cli
mktg init
mktg doctor --json

Star the repo if it's useful; open an issue if install friction wastes your time:
https://github.com/MoizIbnYousaf/marketing-cli

You now have the playbook. Ship something small this week, log the learning, and let the next session start smarter.

— mktg
```
