# Welcome / Onboarding Email Sequence — mktg installers

## Goal
Move a new installer from "CLI on PATH" to "brand memory started + one skill shipped + compounding habit."

## Audience
People who installed `marketing-cli` / ran `mktg init` (or signed up as installers). Solo founders and agent builders on Claude Code / Cursor / Codex.

## Voice
Direct, proof-first. Prefer: ship, compound, playbook, agent-native, dry-run, brand memory, orchestrate.  
Avoid: synergy, leverage, disrupt, 10x, unlock growth, revolutionary.

## Sequence map

| # | File | Send | Job |
|---|---|---|---|
| 01 | `email-01.md` | Immediate | Confirm install path, doctor, what "done" looks like |
| 02 | `email-02.md` | Day 1 | Minimum brand memory (voice + positioning + audience) |
| 03 | `email-03.md` | Day 3 | Run one skill end-to-end; dry-run habit |
| 04 | `email-04.md` | Day 5 | `/cmo` orchestration + Studio optional |
| 05 | `email-05.md` | Day 8 | Compounding loop: learnings, assets, document-review |

## Success metrics
- Open rate ≥ 45%
- Click rate ≥ 12% on install/docs/GitHub links
- Reply rate: qualitative setup friction (feed `brand/learnings.md`)

## CTA hierarchy
1. Fill brand files / run a skill (product action)
2. GitHub star + issue feedback (community)
3. Studio only as optional human surface

## Do not
- Promise enterprise CMO replacement
- Claim exclusive "only CLI" category ownership
- Dump all 64 skills in email 1
