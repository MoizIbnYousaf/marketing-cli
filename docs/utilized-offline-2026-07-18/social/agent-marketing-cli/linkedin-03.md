# LinkedIn — 03 · Playbook ≠ scheduler

Hot take for builder-founders:

A social scheduler is not a marketing operating system.

Typefully / Buffer / Postiz are great at **queueing posts**. They are adapters in our world — not the product.

What agents actually need:

| Job | What ships it |
|-----|----------------|
| Consistent voice | `brand/voice-profile.md` |
| Orchestration | `/cmo` routes to the right skill |
| Research in parallel | 6 agents (brand / audience / competitive / review) |
| Launch artifacts | Show HN, directories, SEO briefs, atomized social |
| Human review surface | `mktg studio` (same npm package) |

64 skills. Manifest-backed. Drop-in contract — add a SKILL.md, update the manifest, `mktg update`.

We're allergic to "unlock growth" copy. We ship playbooks.

```bash
npm i -g marketing-cli
```

CLI for the agent. Studio for the human.
